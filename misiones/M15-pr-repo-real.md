# M15 — PR a repo real 🥇

**XP**: 200 · **Dificultad**: Boss final · **Badge**: 🥇 Real Contributor

## Briefing

Hasta acá todo fue "para aprender". Esta misión es la primera contribución a un repo de verdad de la empresa. Va a producción. La van a usar otros devs. Va con tu nombre en el commit log para siempre.

## Objetivo técnico

Resolver 1 issue real asignada por el game master en un repo interno. Issue chica, segura, pero real.

## Criterios de aceptación

- [ ] Issue asignada en repo real (no este pasaporte)
- [ ] Branch siguiendo convención del repo (preguntá antes)
- [ ] Cambio mínimo necesario (no refactor de paso, no scope creep)
- [ ] Conventional commit
- [ ] PR description usando template del repo (si existe)
- [ ] PR pasa CI (si hay)
- [ ] PR review aprobado por al menos 1 dev senior
- [ ] PR mergeado a `main` o branch de release
- [ ] Después de merge: chequeás que no rompió nada en producción

## Reglas importantes

- ⚠️ Antes de tocar nada: leé el README del repo, el CONTRIBUTING.md si hay
- ⚠️ Antes de empezar: clarificá con quien asignó qué se espera exactamente
- ⚠️ Si encontrás algo "raro" pero no es tu issue: anotalo, no lo toques
- ⚠️ Si el review pide cambios: agradecé, no te ofendas
- ⚠️ Si te trabás >2 horas: pedí ayuda, no penses que es muestra de debilidad

## Para el PR (description)

Seguí el template del repo. En general:
1. Issue que cierra (`Closes #123`)
2. Qué cambia
3. Cómo probaste
4. Screenshots si es UI

## Reflexión post-merge

En tu pasaporte, agregá entrada en `docs/m15-reflexion.md`:
1. Cómo se sintió ver tu PR mergeado a `main`
2. Diferencia con los PRs de juguete que hiciste antes
3. Una cosa que el code review te enseñó

## Premio

🥇 Badge dorado. El más importante del pasaporte. Ponelo grande en el README.
