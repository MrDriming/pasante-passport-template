# Boss 3 — Deploy + README pro + demo 🏆

**XP**: 100 · **Dificultad**: Boss · **Badge**: 🏆 Shipped It

## Briefing

Tu app está hecha. Falta dejarla **profesional**: README que cualquiera entienda, deploy estable, demo al equipo. Esto es lo que diferencia un proyecto de portfolio de un proyecto "de la escuela".

## Objetivo técnico

App pulida en Vercel + README pro + demo de 5 minutos al equipo.

## Criterios de aceptación

### Deploy
- [ ] URL `.vercel.app` estable (sin errores en producción)
- [ ] Domain custom si querés (opcional)
- [ ] Build sin warnings
- [ ] Funciona en desktop y mobile (probá en tu celular real)
- [ ] No hay `console.log` de debug ni alerts dejados

### README pro
- [ ] Título + 1 oración descripción
- [ ] Badges (Vercel, lenguaje, license — usá [shields.io](https://shields.io))
- [ ] Screenshot o GIF de la app andando
- [ ] Sección "Features" con lista
- [ ] Sección "Stack" (qué tecnologías)
- [ ] Sección "Cómo correr local" con pasos exactos (`git clone`, `npm install`, `npm run dev`)
- [ ] Sección "Cómo se hizo" — 1 párrafo sobre el proceso
- [ ] Link a la demo en vivo

### Demo (5 min)
- [ ] Slides cortos o solo la app
- [ ] Mostrás el flujo principal en vivo
- [ ] Contás 1 cosa que aprendiste haciéndolo
- [ ] Contás 1 cosa que no funcionó como esperabas
- [ ] Aceptás 2-3 preguntas

## Hints

- Para screenshot animado: [ScreenToGif](https://www.screentogif.com/) o LICEcap
- README con buen formato: mirá repos populares en GitHub como referencia (ej: [next.js](https://github.com/vercel/next.js))
- Antes de demo: corré la app en otro browser/incógnito por si hay bugs solo en tu sesión
- No leas las slides. Hablá.

## Para el PR

PR final del README + ajustes. Description con:
1. Link a producción
2. Lo que más te enorgullece
3. Lo que cambiarías si fuera v2
