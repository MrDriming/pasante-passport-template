# M11 — App propia: diseño y mockup 🔵

**XP**: 25 · **Dificultad**: Media · **Badge**: 🔵 Designer

## Briefing

Empieza tu app propia. 4 misiones para construirla. Esta es la primera: pensar qué hacer y mockearlo, antes de tocar código.

## Objetivo técnico

Elegir una idea, validarla con el game master, hacer mockup visual.

## Criterios de aceptación

- [ ] Idea aprobada por el game master (alcance realista para 4 sesiones)
- [ ] README del repo con:
  - Nombre de la app
  - 1 párrafo: qué hace y para quién
  - Lista de features (mínimo 3, máximo 6)
  - Stack: Next.js + lo que decidas
- [ ] Mockup de pantallas:
  - Opción A: v0.dev (genera UI con prompts)
  - Opción B: Figma (manual)
  - Opción C: dibujo en papel + foto (válido)
- [ ] Mínimo 2 pantallas mockeadas
- [ ] Subido todo al repo `mi-app-{tu-nombre}`

## Ideas si no se te ocurre nada

- App de partidos de fútbol del finde (cargás partidos, ves quién va)
- Tracker de libros leídos en el año
- Generador de excusas para no ir a un cumple
- Wishlist compartida para regalos
- Random picker para "qué comemos hoy"
- Tracker de hábitos (vasos de agua, horas de gym)

## Restricciones

- Tiene que tener al menos 1 form de entrada de datos
- Tiene que tener al menos 1 lista o vista de datos
- **No backend en M11-M13**. Solo localStorage si guardás algo. Backend opcional para M14.

## Para el PR

PR del README + mockups. Descripción:
1. Por qué elegiste esta idea
2. Para quién la harías si fuera real
3. Qué es lo más difícil que ves venir
